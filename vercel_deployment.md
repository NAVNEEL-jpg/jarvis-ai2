# J.A.R.V.I.S. Vercel Deployment & Phone Remote Access Guide

This guide explains how to deploy the J.A.R.V.I.S. Core Dashboard and Mobile Remote UI to **Vercel** so you can access them on your phone from anywhere (over mobile data) without being connected to your home WiFi.

The background server and VirtualBox VMs will continue to run natively on your laptop. The Vercel front-end UI will talk directly to your laptop backend securely via a public HTTPS tunnel.

---

## 1. Exposing the Laptop Server (Tunnelling)

Since your laptop is behind a home router/NAT, Vercel (which runs in the cloud) cannot reach it directly. You must create a secure, public HTTPS tunnel to your Flask backend (running on port `5000` or `5001`).

### Option A: Using Ngrok (Recommended)
1. Register a free account at [ngrok.com](https://ngrok.com).
2. Install ngrok on your laptop:
   ```bash
   .venv\Scripts\pip install pyngrok
   ```
3. Expose the dashboard backend (running on port `5000`):
   ```bash
   ngrok http 5000
   ```
4. Copy the secure forwarding URL generated by ngrok (e.g., `https://a1b2-34-56-78.ngrok-free.app`).

### Option B: Using Localtunnel (Free, no account needed)
1. Install Node.js on your laptop.
2. Expose port `5000` using localtunnel:
   ```bash
   npx localtunnel --port 5000
   ```
3. Copy the URL provided (e.g., `https://some-subdomain.localtunnel.me`).

---

## 2. Deploying to Vercel

Vercel can host your static folder (`dashboard/static`) directly:

1. The project has a `vercel.json` configuration file at the root of your project directory (`d:\jarvis ai2\vercel.json`):
   ```json
   {
     "version": 2,
     "builds": [
       {
         "src": "dashboard/static/**",
         "use": "@vercel/static"
       }
     ],
     "routes": [
       {
         "src": "/(.*)",
         "dest": "/dashboard/static/$1"
       }
     ]
   }
   ```
2. Commit and push the changes to your GitHub repository `jarvis-ai2`.
3. Log in to [vercel.com](https://vercel.com) using your GitHub account.
4. Click **Add New** -> **Project**.
5. Select your `jarvis-ai2` repository.
6. Under **Build & Development Settings**, if the build fails, configure the **Root Directory** setting to `dashboard/static` in the project settings.
7. Click **Deploy**. Vercel will build and host your dashboard on a public URL (e.g., `https://jarvis-ai2.vercel.app`).

---

## 3. Linking the Vercel Frontend to the Laptop Backend

When you open the Vercel URL on your mobile phone:

1. The page will load and show the black secure lock screen.
2. In the top-right corner of the lock screen, tap the **`LINK UPLINK`** button.
3. Paste the public tunnel URL you copied in Step 1 (e.g., `https://your-tunnel.ngrok-free.app`).
4. Click **OK**. The page will reload and link securely to your local laptop server!
5. J.A.R.V.I.S. will now perform automatic face scans and allow passcode inputs on your phone, communicating directly with the background processes running on your laptop.

---

## 4. Mobile Remote UI (`web_control.py`) on Vercel

If you want to host the mobile remote controller UI:
- The mobile remote controller is also served statically from Vercel!
- Open your Vercel URL on your mobile phone, enter your tunnel URL in **`LINK UPLINK`**, and control the server natively!
